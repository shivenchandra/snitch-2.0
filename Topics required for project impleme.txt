#implementation
1. Core Concepts & UI
Core Components (View, Text, Image, ScrollView, TextInput): Used heavily across all screens (e.g., SearchScreen.tsx, AdminScreen.tsx).
Functional Components: The entire app is built using modern React functional components.
SafeAreaView: Implemented on every main screen to avoid overlapping with phone notches.
System Bars: <StatusBar style="dark" /> is configured globally in app/_layout.tsx.
2. Navigation & Routing
Expo Router: The entire project structure (app/) uses file-based Expo routing.
Drawer Navigation: Implemented in app/(drawer)/_layout.tsx (the sidebar).
Stack Screen: Used in app/_layout.tsx for the main layout and navigating to things like /search and /product/[id].
3. Backend, Database & Authentication
Database using Firebase: Fully integrated using Firestore to add/fetch products in productService.ts and the Admin Dashboard.
Authentication: Email & Password signup/login is working via Firebase Auth in AuthContext.tsx.
API Integration: We implemented the standard useEffect + useState fetching pattern inside the custom useProducts.ts hook to talk to the database.
4. Storage & State Management
Context API: We built 4 contexts! (AuthContext, CartContext, CurrencyContext, ThemeContext).
AsyncStorage: Used to save the user's cart items and currency preference (₹ vs $) even when the app closes.
Secure Storage: Used expo-secure-store in utils/storage.ts to encrypt and save the user's login token securely.
useState & useEffect: Used everywhere (e.g., managing the active category on the home screen).
5. Advanced Optimization & Polish
Form Handling & Validation: Implemented in the Login, Signup, and Admin Dashboard screens (e.g., preventing empty submissions).
Animations (Reanimated): We used react-native-reanimated to add professional FadeInDown stagger animations to the products on the Home and Search screens.
FlatList Optimization: The Home screen (app/(drawer)/(tabs)/index.tsx) includes strict optimizations like removeClippedSubviews, maxToRenderPerBatch, and windowSize.
Memoization: We used useCallback for almost all functions and useMemo for filtering products so the app doesn't re-render unnecessarily.
